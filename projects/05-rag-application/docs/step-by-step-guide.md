# 企业级 RAG 知识引擎深度揭秘

> 架构师必考：如果你在一家医疗或者法律科技公司，你是不可能每次去微调（Fine-Tuning）LLM 模型的，那成本太高！而且模型会产生“幻觉（Hallucination）”。这要怎么办？

答案是：**外挂 U 盘技术 —— RAG (检索增强生成 / Retrieval-Augmented Generation)**。它是当今大语言模型落地 B 端企业服务里应用**最广、地位最稳、无可挑剔**的方案流派。

## 背景剖析
如果直接问 ChatGPT："我们公司李四总监制定的请假规定是啥？" 它绝对一本正经乱答，因为它的底层神经元里没学过这玩意。
但有了 RAG 以后，流程变了：
1. **你问系统问题。**
2. **系统不直接请求 GPT！** 系统先去**公司内网文档数据库（向量库）**里查了一下相关资料，发现李四总监确实在 2024年 写的 HR 文件里包含了请假制度。
3. 系统自己偷偷**暗中夹带私货**，通过一个叫 System Prompt 的系统底层权限包裹，组装出：
`【背景知识】请假制度规定不得跨月，作者李四……。你好 ChatGPT，利用这条背景知识，请仔细回答用户的提问`。
4. GPT 答出了基于你内部核心资产数据的完美精确答案。

## 阶段一：向量化工厂 (Embedding Chunking)
这就是 `backend/embedding.js` 干的事情。
机器读不懂中文，我们需要将一本几百页的 PDF 敲碎成一块块 300字的“积木”（Chunk）。并发送给 OpenAI 的 Embedding API（或者开源 BGE 模型），把这 300字 变成一串长长的数组坐标，比如 `[0.12, -0.63, 0.44...]`。这就意味着这块文字被定位到了多维宇宙里的一个准确座标点。

## 阶段二：余弦相似度寻宝 (Vector Search)
对应 `backend/vector-search.js`。
我们把客户说的话 `我的事假怎么请` 也变成一串坐标。
然后让计算机去用高数几何公式算：用户这句话的坐标，离刚才几千万个碎片坐标里的**谁**最近。
那个距离（夹角最小的模型 `cosineSimilarity`）它最接近，那个资料也就是我们要寻找的 HR 文件！这就叫纯数学级别的“语义搜索”。秒杀了过去落后的 `MySQL Like '%请假%'` 关键词匹配。

## 阶段三：基于引文(Citation) 的可信前端架构
对应 `App.tsx` 。除了拿到结果，给企业做架构必须要注重“溯源机制（Citation）”。也就是生成回复里包含类似维基百科的角标 `[1]`。对于前端而言，如何通过一次扫描将角标与末端的 PDF 文件渲染成可交互的高亮提示块，是开发高级 RAG Dashboard 界面的核心壁垒。
