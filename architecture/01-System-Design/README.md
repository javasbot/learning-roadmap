## 系统设计方法论

```
面试回答框架 (30分钟):

1. 需求澄清 (2-3min)
   ├── 功能需求: 核心功能是什么?
   ├── 非功能需求: QPS? DAU? 延迟要求?
   └── 约束: 技术栈? 团队规模?

2. 高层架构 (5min)
   └── 画出核心组件和数据流

3. 关键设计 (15min)
   ├── 数据模型
   ├── API 设计
   ├── 核心算法/策略
   └── 深入1-2个关键点

4. 优化与扩展 (5min)
   ├── 性能优化
   ├── 容灾/降级
   └── 监控/告警

5. Trade-off (2min)
   └── 说明选择的优劣和原因
```
