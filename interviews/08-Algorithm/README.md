# 数据结构与算法面试题

## Q1: 数组 / 链表 / 哈希表面试高频题 [P7]

- **两数之和**：哈希表存储已遍历元素的值和索引，查找 `target - current`（时间 O(n)，空间 O(n)）。
- **合并两个有序链表**：双指针迭代比较，或者递归（时间 O(n+m)，空间 O(1)）。
- **反转链表**：维护 `prev`, `curr`, `next` 指针，逐步翻转（时间 O(n)，空间 O(1)）。
- **环形链表**：快慢指针，相遇说明有环（时间 O(n)，空间 O(1)）。

## Q2: 树和图面试高频题 [P8]

- **二叉树的前中后序遍历 (非递归)**：
  - 前序 / 中序：使用栈 `stack`。
  - 后序：前序遍历改进版（根-右-左），最后结果反转。
- **层序遍历（BFS）**：使用队列 `queue`，记录每层 `size`。
- **二叉树的最近公共祖先 (LCA)**：递归，左边找到一个，右边找到一个，当前即为祖先；如果都在一侧则继续递归（时间 O(n)）。

## Q3: 排序和查找 [P7]

- **快速排序 (Quick Sort)**：
  选取基准（pivot），分为小于和大于的两部分，递归排序，最后合并。平均 O(n log n)，最坏 O(n^2)。
- **二分查找 (Binary Search)**：
  需要在有序数组中查找。维护 `left` 和 `right`，注意 `mid` 防溢出 `mid = left + Math.floor((right - left) / 2)`。

## Q4: 动态规划 (DP) [P8]

- **爬楼梯**：`dp[n] = dp[n-1] + dp[n-2]`，可以用两个变量压缩空间为 O(1)。
- **最长递增子序列 (LIS)**：`dp[i] = max(dp[j]) + 1`，也可以用二分替换 DP 达到 O(n log n)。
- **背包问题**：`dp[i][j]` 表示前 i 个物品重量为 j 时的最大价值。

## Q5: 滑动窗口与双指针 [P8]

- **无重复字符的最长子串**：双指针 `left` 和 `right`。`right` 移动扩张窗口，若有重复字符则 `left` 跳跃到重复字符的下一位（利用 Map 记录位置）。
- **三数之和**：先排序，固定一个数 `nums[i]`，然后用双指针在 `i+1` 到 `length-1` 中寻找两数之和等于 `-nums[i]`，注意去重。

## 前端特有算法要求

相比于后端，前端 P8/P9 面试中纯算法题的比重逐渐降低，更多结合实际业务：
- **树结构转换**：平铺的 Array（带 parentId）转化为嵌套的 Tree 结构。
- **DOM diff 算法**：React Fiber 中的链表遍历和 Diff 启发式策略（LIS求最长递增子序列优化 DOM 移动）。
- **LRU Cache**：利用 Map 的插入有序特性，或手写双向链表+哈希表实现 P9 级缓存系统。
- **大数运算**：字符串模拟加法 / 减法，处理 JS 超出安全范围的整数精度问题。
